# Асинхронный REST-сервис на Python, который:
- получает **текущую погоду** по координатам (Open-Meteo),
- позволяет **добавлять города** в отслеживаемые,
- хранит **прогноз на текущий день** для отслеживаемых городов,
- **обновляет прогноз каждые 15 минут** автоматически,
- отдаёт прогноз **на сегодня в указанное время** и **по выбранным полям**,
- (дополнительно) поддерживает **пользователей**: у каждого пользователя свой список городов + управление им.


## Стек технологий
- Python 3.11+ (рекомендуется 3.12)
- FastAPI + Uvicorn
- httpx (async)
- SQLite + aiosqlite
- Open-Meteo Forecast API
- Open-Meteo Geocoding API (поиск городов)

## API
### GET /health
- проверяет работоспособность сервиса

### POST /cities
- добавляет город в отслеживаемые (сохранить в базу данных) и сразу сохраняет прогноз на сегодня.

Body (JSON):
name (str) - название города
lat (float) - широта
lon (float) - долгота

Пример:
{"name":"Moscow","lat":55.75,"lon":37.61}

Возможные ошибки:
422 — тело запроса не прошло валидацию (пустое имя, lat/lon вне диапазона)
409 — город уже существует
502 — Open-Meteo недоступен/вернул ошибку

### GET /cities
- возвращает список всех отслеживаемых городов из базы данных

Пример:
/cities

### GET /cities/catalog
- выполняет поиск городов через Open-Meteo Geocoding API

Query-параметры:
q (str) — поисковая строка (часть названия)
count (int, optional) — сколько результатов вернуть 
language (str, optional) — язык результата 

Пример:
/catalog/cities?q=moscow&count=10&language=ru

Возможные ошибки:
400 — пустой q, неверный count
502 — ошибка geocoding API

### GET /weather/at
- возвращает прогноз на сегодня для указанного города в указанное время и по выбранным полям. Берёт прогноз из базы данных. Если прогноз уже сохранён, не делает запроса в open-meteo

Query-параметры:
city (str) — название города 
time (str) — время: HH:MM (сегодня) или YYYY-MM-DDTHH:MM
fields (str, optional) — поля через запятую:
temperature
humidity
wind_speed
precipitation
Если fields не задан, то возвращаются все поля.

Примеры:
/weather/at?city=Moscow&time=14:00
/weather/at?city=Moscow&time=14:00&fields=temperature,wind_speed

Возможные ошибки:
400 — пустой city/time, неправильный формат time, неизвестные fields
404 — Город не найден
404 — Город есть, но прогноз ещё не сохранён
404 — Такого времени нет
500 — некорректные данные hourly в БД 

### GET /weather/current
- получает текущую погоду по координатам из open-meteo

Query-параметры:
lat (float) — широта [-90..90]
lon (float) — долгота [-180..180]

Пример:
/weather/current?lat=55.75&lon=37.61

Возможные ошибки:
422 — нет/неверные lat/lon
502 — ошибка обращения к Open-Meteo

### GET /users/{user_id}/weather/at
- возвращает прогноз на сегодня для города из списка пользователя в указанное время и по выбранным полям

Query-параметры:
city (str) — название города
time (str) — время: HH:MM (сегодня) или YYYY-MM-DDTHH:MM
fields (str, optional) — поля через запятую:
temperature
humidity
wind_speed
precipitation
Если fields не задан, возвращаются все поля.

Примеры:
/users/1/weather/at?city=Moscow&time=14:00
/users/1/weather/at?city=Moscow&time=2026-03-02T14:00&fields=temperature,wind_speed

Возможные ошибки:
400 — пустой city/time, неправильный формат time, неизвестные fields
404 — пользователь не найден
404 — город не найден
403 — город существует, но не добавлен пользователю
404 — город есть, но прогноз ещё не сохранён
404 — такого времени нет в прогнозе на сегодня
500 — некорректные данные hourly в БД

### POST /users/{user_id}/cities
- добавляет город в список отслеживаемых конкретного пользователя

Body (JSON)
I вариант:
name (str) — название города
lat (float) — широта
lon (float) — долгота

II вариант (если город уже есть в общем списке):
name (str) - название города 

Возможные ошибки:
404 — пользователь не найден
422 — тело запроса не прошло валидацию 
409 — город уже добавлен пользователю
502 — Open-Meteo недоступен/вернул ошибку 
500 — ошибка БД

### GET /users
- выводит список всех существующих пользователей

Пример:
/users

Возможные ошибки:
500 - ошибка базы данных

### POST /users/register
- регистрация пользователя

Body (JSON):
name (str) — имя пользователя

Пример:
{"name": "Kollessky"}

Возможные ошибки:
422 — имя пользователя не прошло валидацию 
409 — пользователь уже существует 
500 — ошибка базы данных

###  DELETE /user/{user_id}
- удаляет пользователя по указанному user_id

Пример:
DELETE /users/1

Возможные ошибки:
404 - пользователь не найден
500 - ошибка базы данных

### DELETE /user/{user_id}/cities/{city}
- удаляет город из списка пользователя 

Path-параметры:
user_id (int) - id пользователя
city (str) - название города

Пример:
DELETE /users/1/cities/Moscow

Возможные ошибки:
404 — пользователь не найден
404 — город не найден
404 — город существует, но не привязан к пользователю
500 — ошибка БД
